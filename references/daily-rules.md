# 互联网英语日报 生成规范

> 本文档是日报生成规则的核心文档，每次生成日报前必须读一遍。用户可随时 review 确保规则被正确执行。

---

## 一、数量要求（固定，不可增减）

| 内容 | 数量 | 说明 |
|------|------|------|
| 岗位术语 | 3个 | 每个含名称（缩写）+ 中文释义 |
| 场景句 | 3个 | 每句含英文表达 + 关键词中英标注 |
| AI热词 | 2个 | 实时生成，结合当前日期动态引入 |

---

## 三、术语选择规则（按优先级从高到低）

### 3.1 跨类分散
每次选的3个术语必须来自**不同子分类**。例如 ops 词库有6个子分类，每次从其中3个各选1个。

ops 词库子分类：
- 一、用户运营
- 二、活动运营
- 三、流量与渠道
- 四、内容与社区运营
- 五、指标与分析
- 六、工具与流程

### 3.2 低频高词优先
**以下高频烂大街词每次最多选1个，且间隔至少2周才能再出现：**
DAU、MAU、GMV、ROI、ROAS、Retention、Churn、CAC、LTV、CTR、CPM

优先选相对陌生但有价值的术语，如 Cohort analysis、Win-back rate、Acquisition Funnel、Sentiment、Share of voice、Seed content、Moderation 等。

### 3.3 近期去重
从 `sent-log.json` 读取该用户最近14天已发送的术语记录，**本次不重复出现**。生成日报后，将本次发出的术语追加写入 `sent-log.json`。

### 3.4 例句必须不同
同一术语在不同期次出现时，**例句必须全部重新生成**，不得重复使用历史例句。

## 四、场景句规则

### 4.1 来源
从 `references/scenes/` 下当日对应场景文件中随机抽取3个表达亮点，深度展开。

场景轮换表：
| 周期 | 主打场景 | 对应文件 |
|------|----------|----------|
| 周一 | 商务邮件 | email.md |
| 周二 | 会议发言 | meeting.md |
| 周三 | 向上沟通 | managing-up.md |
| 周四 | 述职汇报 | presentation.md |
| 周五 | Small Talk | small-talk.md |
| 周末 | 复习总结 | review.md |

### 4.2 黑话融入场景
各场景文件末尾均有「职场黑话」专区，黑话按场景归属分发。生成场景句时，**优先从该场景的黑话区选取表达**，让用户自然学到"对齐颗粒度用 align on granularity"、"推不动用 hit a wall"这类高频黑话。

- meeting.md 黑话：对齐、拉通、过一下、盘、穿透、收敛、先hold、推不动
- managing-up.md 黑话：排期、兜底、跑起来、端到端、先跑通、推不动
- presentation.md 黑话：定稿、归因、打法、抓手、扩规模、上线
- new-hire-icebreak.md 黑话：验证假设、先占坑、借力、赋能、提效、闭环、先POC、先跑通

### 4.3 关键词中英标注
每个场景句后面，用 `→ {关键词}（{中文翻译}）` 格式标注2~3个关键词，每个关键词必须有中文翻译，不得只写英文。

---

## 五、AI热词规则

每次从 `references/terms/ai-trends.md` 基准词库中选2个，结合当前日期动态生成新例句，带入当季最新表达。无需依赖文件本身的定期更新，模型实时生成。

---

## 六、用户订阅状态

读取 `references/subscriptions.json` 获取用户订阅的频道列表（mis_id → 频道数组）。每个频道独立生成，互不重复。

---

## 七、生成后操作

生成日报后，将本次发出的3个术语追加写入 `references/sent-log.json`，格式：
```json
{
  "ops": {
    "2026-04-21": ["术语1", "术语2", "术语3"],
    "2026-04-20": ["术语4", "术语5", "术语6"]
  }
}
```

## 维护说明

本文档是日报生成规则的唯一真实来源（SKILL.md 里的规则描述为简化版，以本文档为准）。

**用户 Review 方式**：每次打开本文件，逐条对照最近收到的日报，检查是否符合规则。如有不符合，在文件中标注或告知 agent 修改。

**规则更新场景**：
- 日报数量变化（如增减术语数量）→ 改本文档一
- 术语选择规则调整（如新增低频词清单）→ 改本文档三
- 新增场景文件或轮换表调整 → 改本文档四
- sent-log 路径或格式调整 → 改本文档三和七
